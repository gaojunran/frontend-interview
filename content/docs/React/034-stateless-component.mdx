---

## 1. 什么是无状态组件？

在 React 里，无状态组件指的是 **不维护自己的内部状态（state）** 的组件。它们只依赖于传入的 `props` 来渲染 UI，不会自行存储和管理数据状态。

简单来说，无状态组件：

* 没有自己的 `this.state` 或 `useState` 等状态管理
* 不涉及生命周期中的状态变化
* 通常只负责接收 props 并返回 JSX，纯粹展示数据

### 两种典型写法：

#### a. 函数组件（Function Component）

```jsx
function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}
```

这里的 `Greeting` 组件只根据传入的 `name` 来渲染，没有任何状态。

#### b. 纯函数式组件（Pure Functional Component）

也就是严格意义上的无状态组件，只依赖输入（props），不会引起副作用，也不会修改外部变量。

---

## 2. 为什么叫“无状态组件”？

“无状态”强调这个组件本身不持有任何状态数据，它只是一个纯展示组件。它的输出（UI）只依赖输入（props），同样的输入，永远输出同样的内容。

---

## 3. 无状态组件的优势

* **简单易维护**：代码更简洁，逻辑单一，只有渲染相关，不涉及状态管理。
* **性能优化**：无状态组件一般不会自己触发重新渲染（只有 props 变化时才会更新），有助于避免不必要的渲染。
* **可测试性强**：纯函数特性，输入和输出确定，容易写单元测试。
* **更容易复用**：职责单一，专注 UI 展示，适合作为子组件复用。

---

## 4. 无状态组件的应用场景

无状态组件通常用来：

### 4.1 纯展示组件（UI 组件）

比如按钮、图标、列表项、卡片、标题等，只负责展示传入的数据和样式，不参与逻辑和状态。

```jsx
function Button(props) {
  return <button onClick={props.onClick}>{props.label}</button>;
}
```

### 4.2 作为受控组件的子组件

配合父组件管理状态，子组件仅接收数据和事件回调（props），完成 UI 渲染。比如表单输入项组件。

### 4.3 组合和复用

当页面由多个小组件拼装起来时，那些只负责显示、样式的部分适合用无状态组件，提升结构清晰度。

### 4.4 性能敏感场景

由于无状态组件容易被 `React.memo` 等工具优化，适合用于频繁渲染且更新逻辑简单的组件。

---

## 5. 对比有状态组件（Stateful Component）

有状态组件维护自己的 `state`，管理数据变化和生命周期，负责业务逻辑、交互处理。

```jsx
class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  increment = () => {
    this.setState({ count: this.state.count + 1 });
  };

  render() {
    return <button onClick={this.increment}>{this.state.count}</button>;
  }
}
```

无状态组件更多偏向展示，有状态组件更多负责行为。

---

## 6. 小结

| 特点   | 无状态组件     | 有状态组件              |
| ---- | --------- | ------------------ |
| 状态   | 无内部状态     | 有自己的 state         |
| 代码风格 | 纯函数式写法，简单 | class 或 useState 等 |
| 关注点  | 只负责 UI 渲染 | 管理数据和业务逻辑          |
| 适用场景 | 纯展示、复用组件  | 交互、数据状态变化管理        |
| 性能   | 容易优化，渲染轻量 | 复杂组件可能性能开销大        |
