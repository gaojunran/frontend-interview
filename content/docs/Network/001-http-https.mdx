---
title: 什么是HTTP? HTTP 和 HTTPS 的区别?
---

HTTP（HyperText Transfer Protocol，超文本传输协议）是**Web 上通信的基础协议**，用来在客户端（通常是浏览器）和服务器之间传输超文本（如 HTML 页面、图片、视频、JSON 数据等）。

---

## 一、什么是 HTTP？

### 1. 基本概念

HTTP 是一种**无状态**、**面向请求/响应**的协议，通常运行在 TCP 协议之上（默认端口 80）。

* **无状态**：每次请求之间互相独立，服务器不会记住上次的请求。
* **请求/响应模式**：客户端发送请求（Request），服务器返回响应（Response）。
* **明文传输**：数据不加密，任何中间人都可以查看、篡改。

### 2. 常见 HTTP 请求方法

* `GET`：获取资源（无副作用）
* `POST`：提交数据（如表单、JSON）
* `PUT`：更新资源
* `DELETE`：删除资源
* `HEAD`、`OPTIONS`、`PATCH` 等也有特殊用途

### 3. 报文结构（简化版）

**请求报文**

```
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
```

**响应报文**

```
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1024

<html>...</html>
```

---

## 二、什么是 HTTPS？

### HTTPS（HyperText Transfer Protocol Secure）

HTTPS 是 **HTTP + SSL/TLS 加密** 的版本，确保通信内容的机密性、完整性和身份验证。

* 默认端口：443
* 基于 HTTP，但使用 **SSL/TLS 加密协议** 在底层加密
* 被广泛用于登录、支付、电商等场景

---

## 三、HTTP vs HTTPS：核心区别

| 特性      | HTTP          | HTTPS                          |
| ------- | ------------- | ------------------------------ |
| 安全性     | 明文传输，容易被窃听或篡改 | 加密传输，防窃听、防中间人攻击                |
| 端口号     | 默认使用 80       | 默认使用 443                       |
| 加密方式    | 不加密           | 使用 TLS/SSL 进行加密                |
| 身份验证    | 无法确认服务器身份     | 通过数字证书验证服务器是否可信                |
| 性能      | 快（少一步握手）      | 稍慢（需 TLS 握手），但现在优化后差别很小        |
| SEO 友好度 | 普通            | 谷歌等搜索引擎更偏好 HTTPS 网站            |
| 使用证书    | 不需要           | 必须申请数字证书（如 Let’s Encrypt 免费证书） |

---

## 四、HTTPS 工作原理简述

1. 浏览器访问 HTTPS 网站（如 [https://example.com）](https://example.com）)
2. 服务器返回 SSL 证书（包含公钥）
3. 浏览器验证证书是否可信（如是否被 CA 签名）
4. 双方协商加密算法、生成对称密钥（通过非对称加密实现）
5. 建立安全通道后开始加密通信

这一步骤称为 **TLS 握手过程**，是在正式的 HTTP 请求发送之前完成的。

---

## 五、总结

| 特性          | HTTP     | HTTPS         |
| ----------- | -------- | ------------- |
| 是否加密        | 否        | 是             |
| 默认端口        | 80       | 443           |
| 是否验证身份      | 否        | 是（SSL 证书）     |
| 数据是否能被中间人窃听 | 能        | 不太可能          |
| SEO 优势      | 无        | 有             |
| 性能          | 稍快（没有握手） | 稍慢但更安全（握手后加密） |
