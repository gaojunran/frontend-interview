---
title: 说一下 GET 和 POST 的区别？
---

`GET` 和 `POST` 是 HTTP 协议中最常用的两种请求方法，它们都用于客户端与服务器之间的数据交互，但在**语义、数据传输方式、安全性、缓存机制、使用场景**等方面有显著区别。

---

## ✅ 1. **语义区别**

| 方法       | 含义                    |
| -------- | --------------------- |
| **GET**  | 从服务器获取资源（Read）        |
| **POST** | 向服务器提交数据或创建资源（Create） |

---

## ✅ 2. **参数传递方式**

| 对比点          | GET                                       | POST                    |
| ------------ | ----------------------------------------- | ----------------------- |
| **参数位置**     | 通过 URL 的查询字符串（`?key=value&key2=value2`）传递 | 放在 **HTTP 请求体（body）** 中 |
| **URL 长度限制** | 有（受浏览器或服务器限制，通常约 2KB\~8KB）                | 理论上无限制（受服务器配置限制）        |
| **明文/加密**    | 明文（URL可见，容易被记录）                           | 请求体传输，稍微安全些（但本质上也不是加密）  |

---

## ✅ 3. **是否幂等**

* **GET 是幂等的**（多次调用不会产生副作用）
* **POST 是非幂等的**（多次调用可能创建多个资源）

例子：

```http
GET /api/user?id=1   # 获取用户信息，不会修改数据
POST /api/user       # 创建一个用户，每次提交都可能新增数据
```

---

## ✅ 4. **是否可被缓存**

| GET                  | POST            |
| -------------------- | --------------- |
| 默认可以被浏览器缓存（如搜索结果、图片） | 不会被自动缓存（除非手动配置） |

---

## ✅ 5. **安全性（敏感数据）**

| GET               | POST |
| ----------------- | ---- |
| **安全性较低**：URL 中暴露敏感信息，如密码 | **稍微安全些**，参数在请求体中 |
| 日志、历史记录、书签 都会记录 URL       | 请求体不会被浏览器记录       |

⚠️ **注意**：**GET 和 POST 本质上都不安全**，要依靠 HTTPS 才能真正保护数据传输安全。

---

## ✅ 6. **常见使用场景**

| 使用场景             | 方法                                     |
| ---------------- | -------------------------------------- |
| 查询数据（如搜索、获取用户信息） | `GET`                                  |
| 提交表单（注册、登录、上传文件） | `POST`                                 |
| Ajax 提交数据        | 一般用 `POST`                             |
| RESTful 接口调用     | `GET`、`POST` 结合使用（还有 `PUT`、`DELETE` 等） |

---

## ✅ 7. 举例对比

```http
GET /search?q=chatgpt&page=1
# 查询参数显示在 URL 中

POST /login
Content-Type: application/x-www-form-urlencoded
Body:
username=admin&password=123456
# 参数放在请求体中，不出现在 URL
```

---

## ✅ 8. 总结表格

| 比较维度     | GET       | POST         |
| -------- | --------- | ------------ |
| 请求语义     | 获取资源      | 提交数据         |
| 参数位置     | URL 查询字符串 | 请求体          |
| URL 长度限制 | 有限制       | 无限制          |
| 安全性      | 较低        | 稍好，但仍需 HTTPS |
| 是否缓存     | 可缓存       | 默认不缓存        |
| 幂等性      | 是         | 否            |
| 常用场景     | 查询、获取数据   | 提交表单、上传数据    |
