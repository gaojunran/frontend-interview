---
title: TypeScript 中 void, never, any, unknown 有什么区别？
---

在 TypeScript 中，`void`、`never`、`any` 和 `unknown` 是几种特殊的类型，它们在类型系统中扮演着非常不同的角色。我们来详细对比它们：

---

## 一览对比

| 类型        | 含义                  | 是否可赋值任何值 | 是否可赋值给任意类型           |
| --------- | ------------------- | -------- | -------------------- |
| `void`    | 表示没有返回值（通常用于函数返回类型） | 否        | 可以（通常赋给 `undefined`） |
| `never`   | 表示永远不会有值（抛错/死循环）    | 否        | 可以（是所有类型的子类型）        |
| `any`     | 表示任意类型，关闭类型检查       | 是        | 是                    |
| `unknown` | 表示不确定类型，需先缩小类型再使用   | 是        | 否（必须做类型缩小）           |

---

## 1. `void`

### 含义：

表示函数没有返回值，或一个变量只能是 `undefined`。

### 常见用法：

```ts
function logMessage(message: string): void {
  console.log(message); // 没有 return 值
}
```

### 注意事项：

* 可以把 `undefined` 赋给 `void` 类型变量。
* 用于限制函数不能有返回值。
* 通常不推荐作为变量类型。

---

## 2. `never`

### 含义：

表示永远不会有返回值的类型，比如函数抛异常或死循环。

### 常见用法：

```ts
function throwError(message: string): never {
  throw new Error(message);
}

function infiniteLoop(): never {
  while (true) {}
}
```

### 特点：

* `never` 是所有类型的子类型。
* 没有任何类型是 `never` 的子类型（除了 `never` 自己）。
* 可以用于类型收窄中作为“不可能”的分支检查。

---

## 3. `any`

### 含义：

任意类型，不进行任何类型检查，相当于关闭了类型系统的保护。

### 常见用法：

```ts
let anything: any = 'hello';
anything = 123;
anything = () => {};
```

### 特点：

* 可以赋值为任意类型。
* 也可以赋值给任何其他类型（极不安全）。
* **不推荐使用**，除非需要兼容老代码或逐步迁移。

---

## 4. `unknown`

### 含义：

表示未知类型，比 `any` 更安全，使用前需要做类型判断。

### 常见用法：

```ts
let value: unknown = 'hello';

if (typeof value === 'string') {
  // 现在才可以使用它作为字符串
  console.log(value.toUpperCase());
}
```

### 特点：

* 可以接受任何类型的值。
* 不能直接赋值给其他具体类型（需要先缩小类型）。
* 通常用于接口数据、外部 JSON、用户输入等不确定场景。

---

## 额外对比：`any` vs `unknown`

| 特性     | `any`   | `unknown`   |
| ------ | ------- | ----------- |
| 类型安全   | 无       | 有（需类型检查）    |
| 能否调用方法 | 可以（不检查） | 不可以（需先判断类型） |
| 推荐程度   | ❌ 不推荐   | ✅ 推荐用于不确定类型 |

---

## 总结适用场景

| 类型        | 使用场景描述                                 |
| --------- | -------------------------------------- |
| `void`    | 无返回值的函数，例如 `console.log()`             |
| `never`   | 抛异常或永不返回的函数，或做类型穷尽检查（exhaustive check） |
| `any`     | 不想做类型检查（如快速原型、迁移老代码）                   |
| `unknown` | 外部输入数据的初始类型（需做类型判断）                    |
