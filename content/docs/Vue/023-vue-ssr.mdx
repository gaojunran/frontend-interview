SSR（Server-Side Rendering）解决了什么问题，我主要从以下几点来理解：

### 🧩 SSR 解决的问题：

1. **首屏加载速度慢（SEO 和性能）**：

   * 传统的 SPA 应用首屏内容是通过 JS 渲染的，爬虫抓不到内容，对 SEO 不友好。
   * SSR 直接在服务器输出 HTML，让首屏渲染更快，也更利于搜索引擎收录。

2. **白屏时间长、内容加载慢**：

   * SPA 需要等待 JS 下载和执行后，才能看到页面内容，而 SSR 可以提前返回 HTML，提升用户体验。

3. **社交媒体分享问题**：

   * 比如分享到 Twitter/Facebook 时，这些平台也需要读取页面内容生成预览，SSR 更容易提供这些信息。

---

### ✅ 是否做过 SSR 实践？你是怎么做的？

有做过，使用的是 **Vue 的服务端渲染框架 Nuxt.js**，它对 Vue 做了封装，内建支持 SSR，极大简化了配置和开发流程。

### 🔧 SSR 实现方式（以 Nuxt 为例）：

1. **初始化项目**：

   ```bash
   npx nuxi init my-ssr-app
   cd my-ssr-app
   npm install
   npm run dev
   ```

2. **页面目录结构**：

   * 使用 Nuxt 的 `pages` 目录自动生成路由，所有页面默认就支持 SSR。

3. **服务端获取数据**：

   * 可以在页面中使用 `asyncData()` 或 `useAsyncData()` 获取服务端数据，并在渲染时注入页面。

   ```ts
   export default {
     async asyncData({ $axios }) {
       const data = await $axios.$get('/api/posts')
       return { data }
     }
   }
   ```

4. **部署到服务器**：

   * 使用 `npm run build` 打包，`npm run start` 启动生产环境的 SSR 服务。

5. **缓存优化 + CDN 加速**：

   * 配合服务器端缓存（如 Redis）或 CDN，提高 SSR 性能。

---

### 🧠 总结：

* SSR 带来了更好的首屏性能、SEO 和用户体验；
* 实战中我使用的是 Nuxt.js，它简化了 SSR 的实现流程；
* 同时也了解 SSR 的一些挑战，比如服务端压力增加、缓存控制复杂等，需要结合业务权衡使用。
