---
title: Vue3 中 v-cloak 和 v-pre 有什么用？
---

在 Vue 3 中，`v-cloak` 和 `v-pre` 是两个 **特殊指令**，它们主要用于 **模板渲染控制与优化**，下面是它们的详细作用和使用场景：

---

## ✅ `v-cloak`

### **作用：**

防止在 Vue 实例挂载前，插值表达式（如 `{{ message }}`）短暂显示在页面上。

### **使用场景：**

当你使用 Vue 时，如果页面加载较慢，或者 JS 文件未加载完成，可能会看到类似 `{{ message }}` 这样的模板内容一闪而过，这会给用户一种不完整、不专业的感觉。

### **用法：**

```html
<div id="app" v-cloak>
  {{ message }}
</div>

<style>
[v-cloak] {
  display: none;
}
</style>
```

> Vue 会在实例挂载完成后自动移除 `v-cloak` 属性，所以你可以用 `[v-cloak]` 选择器隐藏它，直到 Vue 完全加载。

---

## ✅ `v-pre`

### **作用：**

跳过这个元素和子元素的编译过程，只保留原始的 Mustache 标签内容等。

### **使用场景：**

* 提升性能：当你有一些不需要 Vue 编译的内容，可以使用 `v-pre` 跳过解析。
* 展示模板语法：例如用于文档或教程中显示 `{{ variable }}` 原样。

### **用法：**

```html
<div v-pre>
  {{ this will not be compiled }}
</div>
```

结果渲染到页面会是：

```html
{{ this will not be compiled }}
```

---

## ✅ 总结对比：

| 指令        | 作用             | 场景          |
| --------- | -------------- | ----------- |
| `v-cloak` | 防止页面闪现未编译模板内容  | 页面初始加载优化    |
| `v-pre`   | 跳过模板编译（保留原始内容） | 提升性能或展示模板语法 |

这两个指令不会影响逻辑功能，主要用于 **页面渲染优化或特殊展示需求**。在大型应用中合理使用它们可以提升用户体验。
