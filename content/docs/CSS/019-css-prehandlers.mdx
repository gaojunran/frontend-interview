---

CSS 预处理语言（CSS 预编译语言）是在标准 CSS 之上扩展的一类语言，它提供了变量、嵌套、函数、混入（Mixin）等编程特性，使样式表更强大、模块化、可维护。

---

## 🔍 一、什么是 CSS 预处理语言？

CSS 本身是一种声明式语言，功能有限，尤其在大型项目中容易出现重复代码、难以维护等问题。预处理语言就是在 CSS 基础上，加入编程语言的一些特性（如变量、条件判断、循环等），写好后再“编译”成标准 CSS 文件，供浏览器使用。

---

## 🧰 二、常见的预处理语言

| 名称     | 文件后缀              | 语言特点                                              |
| ------ | ----------------- | ------------------------------------------------- |
| Sass   | `.sass` / `.scss` | 最早的预处理语言，功能强大，有两种语法：缩进式 `.sass` 和类 CSS 语法 `.scss` |
| Less   | `.less`           | 学习曲线低，语法简单，适合初学者                                  |
| Stylus | `.styl`           | 语法灵活，可省略分号、大括号                                    |

---

## 📐 三、核心特性对比

| 特性      | Sass/SCSS        | Less        | Stylus    |
| ------- | ---------------- | ----------- | --------- |
| 变量      | ✅ `$color`       | ✅ `@color`  | ✅ `color` |
| 嵌套规则    | ✅                | ✅           | ✅         |
| Mixin   | ✅                | ✅           | ✅         |
| 函数/运算   | ✅ 自定义函数丰富        | ✅ 函数有限      | ✅ 灵活支持    |
| 导入模块    | ✅ `@use/@import` | ✅ `@import` | ✅         |
| 条件/循环语句 | ✅ `@if`, `@for`  | ✅           | ✅         |
| 插值语法    | `#{$var}`        | `@{var}`    | `{var}`   |

---

## ✅ 四、使用场景优势

* **大型项目结构清晰**：可以模块化拆分样式，提升维护性。
* **样式复用强**：通过 Mixin、函数等方式减少重复。
* **动态样式**：支持变量和运算，可以根据主题变化生成样式。
* **团队协作统一规范**：统一的变量和函数接口方便多人协作。

---

## ⚠️ 五、区别总结（重点）

| 维度      | Sass/SCSS              | Less     | Stylus      |
| ------- | ---------------------- | -------- | ----------- |
| 开发主流程度  | ⭐⭐⭐⭐（SCSS 最常用）         | ⭐⭐⭐      | ⭐⭐          |
| 学习曲线    | 中                      | 低        | 高（语法自由度高）   |
| 编译工具    | dart-sass, node-sass 等 | less.js  | stylus      |
| Node 支持 | ✅                      | ✅        | ✅           |
| 使用语法    | SCSS 类似 CSS，容易上手       | 与 CSS 很像 | 语法自由，可无括号分号 |

---

## 🚧 六、注意事项

* 编译预处理语言需要额外构建工具支持，如 webpack、vite 中配置相应 loader。
* 预处理语言最终还是编译为标准 CSS，不能直接在浏览器中运行。
* 新的 CSS 特性（如：`custom properties` 自定义属性）正逐步减少预处理语言的必要性。

---

## 💡 七、现代趋势

* 随着 CSS 自定义属性（`--var`）、嵌套（CSS Nesting）和 `@layer` 等新特性的支持增强，未来对预处理语言的依赖可能会减少。
* 但在大型项目中，Sass（SCSS 语法）依然是最主流的方案，很多 UI 框架（如 Bootstrap）仍使用。
