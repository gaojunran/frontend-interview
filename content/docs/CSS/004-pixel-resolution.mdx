---
title: 说说设备像素、css像素、设备独立像素、dpr、ppi 之间的区别？
---

这些概念都是在前端开发中与**屏幕显示和分辨率**相关的核心概念，它们虽然相似，但各有侧重。下面我来一一解释并对比它们的区别：

---

## ✅ 1. **设备像素（Device Pixel）**

* **定义**：设备屏幕上的物理像素点（物理小格子）。
* 每个设备像素只能显示一个颜色。
* 数量固定，比如 iPhone 13 的屏幕宽度为 1170 个设备像素。

---

## ✅ 2. **CSS 像素（CSS Pixel / Logical Pixel）**

* **定义**：浏览器中用来布局和渲染的“抽象单位”，不一定对应实际的物理像素。
* 在 CSS 中写的 `width: 100px` 就是 **100 个 CSS 像素**。
* **和设备无关**，设计上让开发人员不用关心设备分辨率的差异。

---

## ✅ 3. **设备独立像素（DIP, Device-Independent Pixel）**

* **定义**：一种抽象单位，设计上希望在不同分辨率设备上保持**视觉一致性**的“理想像素”。
* 在 Web 上，CSS 像素通常等价于 DIP。
* Android/Flutter 里也常用 DIP 表示逻辑像素。

---

## ✅ 4. **DPR（Device Pixel Ratio）**

* **定义**：每一个 CSS 像素对应的设备像素个数。
* 公式：

  ```
  DPR = 设备像素 / CSS 像素
  ```
* 例如：

  * 一个 `width: 100px` 的 div，

    * 在 DPR = 1 的屏幕上，占据 100 个设备像素。
    * 在 DPR = 2 的屏幕上，占据 200 个设备像素。
* 常见值：1（老屏幕）、2（Retina 屏）、3（高端手机）

---

## ✅ 5. **PPI（Pixels Per Inch）**

* **定义**：每英寸的物理像素数量，衡量**屏幕精细程度（分辨率密度）**。
* 单位：像素/英寸
* 数值越高，图像越精细，越不容易看到“颗粒感”。
* 举例：

  * 低端显示器 PPI 约 96
  * MacBook Retina 显示器 PPI ≈ 220+
  * iPhone 13 PPI ≈ 460

---

## 📊 总结对比表

| 名称     | 代表什么     | 单位       | 举例           | 说明                |
| ------ | -------- | -------- | ------------ | ----------------- |
| 设备像素   | 实际物理像素   | px       | 1170×2532    | 真实显示的像素格子         |
| CSS 像素 | 布局用的逻辑像素 | px       | width: 375px | 跟 DPR 有映射关系       |
| 设备独立像素 | 与设备无关的单位 | dp / dip | Flutter 中常见  | 类似 CSS 像素         |
| DPR    | 像素比      | 比值       | DPR=2        | 每个 CSS 像素对应2个物理像素 |
| PPI    | 每英寸像素数量  | px/inch  | 326 PPI      | 数值越大越清晰           |

---

## ✅ 举个例子加深理解：

iPhone 13：

* 分辨率（设备像素）: `1170 x 2532`
* CSS 像素宽度：`390px`（在 Safari 中查看网页）
* DPR = 1170 / 390 = **3**
* PPI = 460+

你设置：

```css
div {
  width: 100px;
}
```

在 iPhone 13 上：

* 实际会占用：**100 × 3 = 300 个物理像素宽度**
* 视觉大小跟普通屏上看到的一样大，但更清晰。
