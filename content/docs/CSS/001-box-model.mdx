---
title: 说说你对盒子模型的理解?
---


CSS 盒子模型（Box Model）是网页布局和设计的基础概念之一，它描述了网页中每个元素是如何占据空间的。每个 HTML 元素在页面中都会被当作一个**矩形盒子**来渲染，这个盒子由以下几个部分组成：

---

### 🌟 盒子模型的四个组成部分（从内到外）：

1. **Content（内容区域）**

   * 实际显示文本、图片等内容的区域。
   * 由 `width` 和 `height` 决定。

2. **Padding（内边距）**

   * 内容和边框之间的空白区域。
   * 设置 `padding` 可以让内容与边框之间留出空间。
   * `padding` 不会影响背景颜色的填充区域（背景会延伸到 padding 区域）。

3. **Border（边框）**

   * 包裹在 padding 外面的边框。
   * 通过 `border` 属性设置颜色、宽度和样式（实线、虚线等）。

4. **Margin（外边距）**

   * 元素之间的间距，不影响元素自身大小。
   * 可以发生“**外边距合并（margin collapse）**”，比如上下两个块级元素之间的垂直间距可能不是两者 margin 的总和，而是取较大者。

---

### 📦 两种盒子模型类型

#### 1. **标准盒子模型（content-box，默认）**

* `width` 和 `height` 只包含内容区域。
* padding 和 border 会额外增加盒子的总宽高。

```css
.box {
  width: 100px;
  padding: 10px;
  border: 5px solid black;
}
```

👉 实际盒子宽度 = `100 + 10*2 + 5*2 = 130px`

#### 2. **IE盒子模型（border-box）**

* `width` 和 `height` 包括了内容、padding 和 border。
* 适合控制元素的总宽高。

```css
.box {
  box-sizing: border-box;
  width: 100px;
  padding: 10px;
  border: 5px solid black;
}
```

👉 内容区域会自动缩小，以保持盒子总宽度为 100px。

---

### 💡 应用建议

* 通常会用：

  ```css
  * {
    box-sizing: border-box;
  }
  ```

  这样更容易控制整体布局，避免盒子大小超出预期。

---

### 🔍 小细节

* `inline` 元素只会应用左右 `margin` 和 `padding`，上下无效。
* `block` 元素才会撑满父容器，才能应用所有方向的边距和内边距。
