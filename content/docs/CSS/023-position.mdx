---
title: 详细讲讲 css 中 position 的值？
---

当然可以！CSS 中的 `position` 属性是控制元素定位行为的核心属性，它决定了元素在页面中的布局方式和参考点。`position` 有几个常用的取值，分别对应不同的定位机制。下面我详细讲讲它们的含义和用法。

---

## CSS `position` 的常见取值

### 1. `static` （默认值）

* **含义**：静态定位，元素按照正常的文档流排列。
* **特点**：

  * 这是所有元素的默认定位方式。
  * 设置 `top`、`left`、`right`、`bottom` 无效。
  * 元素不会脱离文档流。
* **应用**：

  * 不需要特别定位时，通常保持 `static`。

---

### 2. `relative` （相对定位）

* **含义**：相对于元素自身在文档流中的原始位置进行偏移。
* **特点**：

  * 元素仍占据文档流原位置（不会影响周围元素布局）。
  * 可以使用 `top`、`left`、`right`、`bottom` 来偏移元素。
  * 偏移的元素不会脱离文档流。
* **举例**：

  ```css
  .box {
    position: relative;
    top: 10px;  /* 元素向下移动10px */
    left: 5px;  /* 元素向右移动5px */
  }
  ```
* **用途**：

  * 微调元素位置。
  * 为绝对定位的子元素提供定位参照（见后文）。

---

### 3. `absolute` （绝对定位）

* **含义**：元素脱离文档流，相对于最近的已定位祖先元素定位（`position` 不为 `static` 的祖先），如果没有定位祖先，则相对于初始包含块（通常是浏览器视口）定位。
* **特点**：

  * 元素脱离文档流，不占据空间，周围元素表现得好像该元素不存在。
  * 可用 `top`、`left`、`right`、`bottom` 精确控制位置。
* **举例**：

  ```css
  .container {
    position: relative;
  }
  .box {
    position: absolute;
    top: 20px;
    left: 30px;
  }
  ```

  这里 `.box` 会相对于 `.container` 定位，因为 `.container` 是最近的已定位祖先。
* **用途**：

  * 精准定位元素。
  * 弹出菜单、模态框、提示气泡等。

---

### 4. `fixed` （固定定位）

* **含义**：元素相对于浏览器视口固定定位，不随页面滚动移动。
* **特点**：

  * 脱离文档流，不占据空间。
  * 位置固定，滚动页面时保持在视口的固定位置。
  * 也可以使用 `top`、`left`、`right`、`bottom` 控制位置。
* **举例**：

  ```css
  .header {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
  }
  ```
* **用途**：

  * 固定顶部导航栏、回到顶部按钮、固定广告等。

---

### 5. `sticky` （粘性定位）

* **含义**：混合了相对定位和固定定位的行为。元素在跨越指定的滚动阈值前表现为相对定位，超过阈值后表现为固定定位。
* **特点**：

  * 元素在跨越给定阈值之前是相对定位（位置正常，随文档滚动）。
  * 超过阈值后像 `fixed` 一样固定在某个位置。
  * 必须指定至少一个方向属性（`top`、`left`、`right`、`bottom`）才会生效。
  * 定位参照是最近的滚动祖先（而非视口）。
* **举例**：

  ```css
  .header {
    position: sticky;
    top: 0; /* 到达视口顶部时固定 */
  }
  ```
* **用途**：

  * 固定列表标题、导航栏等，滚动时粘住顶部。
* **浏览器支持**：

  * 现代浏览器基本都支持，但部分老旧浏览器不支持。

---

## 总结对比表

| 值          | 是否脱离文档流 | 定位参考       | 支持偏移属性 (`top` 等) | 典型用途        |
| ---------- | ------- | ---------- | ---------------- | ----------- |
| `static`   | 否       | 无（正常文档流）   | 否                | 默认布局        |
| `relative` | 否       | 自身位置       | 是                | 微调位置，定位参照   |
| `absolute` | 是       | 最近已定位祖先或视口 | 是                | 精准定位，弹出层    |
| `fixed`    | 是       | 视口         | 是                | 固定位置，不随滚动移动 |
| `sticky`   | 类似相对+固定 | 最近滚动祖先     | 是                | 粘性定位，滚动时固定  |

---

## 小贴士

* 使用 `absolute` 定位时，建议父元素设置 `position: relative`，这样子元素就会相对于父元素定位，不会跑到视口。
* `fixed` 元素会覆盖页面内容，且不占用空间，需要注意布局。
* `sticky` 是一个强大但不太容易理解的定位方式，适合实现滚动联动效果。
* 如果想让元素在脱离文档流同时又有动画效果，`absolute` 和 `fixed` 是好选择。
