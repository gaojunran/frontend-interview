---
title: 讲讲 CSS Grid，以及应用场景？
---

CSS Grid 是一种强大的二维布局系统，它允许开发者通过“网格”来对页面进行精确的布局控制，适用于同时处理行和列的复杂布局场景。它与 Flexbox 不同，后者更适合一维布局（要么是水平，要么是垂直）。

---

### 一、核心概念

#### 1. **容器属性（父元素）**

* `display: grid;` 或 `display: inline-grid;`
* `grid-template-columns` / `grid-template-rows`：定义列/行数量与宽高，例如：

  ```css
  grid-template-columns: 200px 1fr 1fr;
  grid-template-rows: auto 100px;
  ```
* `grid-gap` / `gap`：网格间距
* `grid-template-areas`：通过命名区域来布局，配合子项的 `grid-area` 使用
* `justify-items`, `align-items`：控制子项在每个网格单元内的对齐方式
* `justify-content`, `align-content`：控制整个网格内容在容器内的对齐方式

#### 2. **子项属性（子元素）**

* `grid-column` / `grid-row`：指定元素跨越的列/行范围

  ```css
  grid-column: 1 / 3; /* 从第一列跨到第三列（不包含） */
  ```
* `grid-area`：与 `grid-template-areas` 配合使用

---

### 二、简单示例

```html
<div class="grid-container">
  <div class="item1">头部</div>
  <div class="item2">侧边栏</div>
  <div class="item3">主内容</div>
  <div class="item4">页脚</div>
</div>
```

```css
.grid-container {
  display: grid;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
  grid-template-columns: 200px 1fr;
  grid-template-rows: auto 1fr auto;
  height: 100vh;
  gap: 10px;
}

.item1 { grid-area: header; }
.item2 { grid-area: sidebar; }
.item3 { grid-area: main; }
.item4 { grid-area: footer; }
```

---

### 三、典型应用场景

#### ✅ **1. 页面整体布局**

用于构建带有头部、侧边栏、主内容、页脚等区域的复杂页面框架，代码语义清晰、可维护性高。

#### ✅ **2. 图片墙 / Gallery 布局**

图片或卡片排列成规则的网格，并可灵活控制哪些元素跨行/跨列。

#### ✅ **3. 表格型排版**

如后台管理系统中的数据面板、控制面板、仪表盘等。

#### ✅ **4. 响应式布局**

结合 `media queries`，可以动态调整网格列数或排列方式，适配不同屏幕尺寸。

---

### 四、Grid vs Flexbox 对比

| 特性   | Grid              | Flexbox       |
| ---- | ----------------- | ------------- |
| 维度   | 二维（行和列）           | 一维（行或列）       |
| 项目顺序 | 可自由定位，不依赖 HTML 顺序 | 顺序严格依赖 DOM 顺序 |
| 复杂布局 | 更擅长复杂整体布局         | 适合单行/单列内容对齐   |
| 对齐方式 | 支持整行/整列/单元格对齐     | 支持主轴/交叉轴对齐    |

---

### 五、小结

> ✅ **Grid 是构建复杂页面布局的首选工具**，尤其当你需要精确控制行列时。
> ✅ **Flexbox 更适合局部布局和内容分布，如导航条、按钮组等。**
> ✅ 它们不是互斥的，**可以组合使用**：Grid 负责框架，Flex 处理局部。
