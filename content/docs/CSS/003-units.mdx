---
title: 说说em / px / rem / vh / vw 区别?
---

在 CSS 中，`em`、`px`、`rem`、`vh`、`vw` 是常见的单位，用于设置字体大小、边距、宽高等。它们的**含义和作用范围不同**，以下是详细的对比和解释：

---

### ✅ 1. `px`（像素）

* **定义**：绝对单位，表示屏幕上的物理像素（在高 DPI 屏幕中是逻辑像素）。
* **优点**：可预测，精确。
* **缺点**：不响应用户设置的缩放或浏览器字体大小设置，不够灵活。
* **示例**：

  ```css
  font-size: 16px;
  width: 200px;
  ```

---

### ✅ 2. `em`

* **定义**：相对单位，**相对于父元素的 `font-size`**。
* **用途**：常用于字体大小、内边距、外边距等。
* **优点**：具有继承性，可实现响应式设计。
* **缺点**：层层嵌套时，容易导致累积计算（例如嵌套两层，`1em` 就变成了父元素的父元素的倍数）。
* **示例**：

  ```css
  body { font-size: 16px; }
  h1 { font-size: 2em; } /* 相当于 32px */
  p { font-size: 0.875em; } /* 相当于 14px */
  ```

---

### ✅ 3. `rem`（Root EM）

* **定义**：相对单位，**相对于根元素（`html`）的 `font-size`**。
* **优点**：避免了 `em` 的嵌套累积问题，更可控、统一。
* **常用于**：全局排版、响应式布局。
* **示例**：

  ```css
  html { font-size: 16px; }
  h1 { font-size: 2rem; } /* 相当于 32px */
  p  { font-size: 1rem; } /* 相当于 16px */
  ```

---

### ✅ 4. `vh`（viewport height）

* **定义**：相对于**视口高度**的 1%。
* **用途**：可用于全屏布局、响应式设计。
* **示例**：

  ```css
  height: 100vh; /* 高度等于整个视口高度 */
  ```

---

### ✅ 5. `vw`（viewport width）

* **定义**：相对于**视口宽度**的 1%。
* **用途**：同上，适合宽度控制。
* **示例**：

  ```css
  width: 50vw; /* 宽度为视口宽度的 50% */
  ```

---

### 🧠 总结对比表：

| 单位    | 相对谁     | 特点           | 用途         |
| ----- | ------- | ------------ | ---------- |
| `px`  | 固定数值    | 精确、不可缩放      | 精确布局       |
| `em`  | 父元素字体大小 | 继承性强，可能层层叠加  | 字体、间距      |
| `rem` | 根元素字体大小 | 全局统一，避免嵌套问题  | 字体、响应式设计   |
| `vh`  | 视口高度    | 随视口变化，适应窗口大小 | 全屏布局、高度    |
| `vw`  | 视口宽度    | 同上           | 宽度适配、响应式布局 |

---

如果你想要做一个**响应式且可访问性好的页面设计**，推荐使用：

* `rem` 作为字体单位；
* `vw/vh` 作为全屏块的单位；
* `px` 用于精细调整；
* `em` 用在局部自适应。
