---
title: web常见的攻击方式有哪些？如何防御？
---

Web 常见的攻击方式主要包括以下几种，每种攻击方式也对应着不同的防御策略：

---

## ✅ 1. **XSS（跨站脚本攻击）**

**攻击原理：**
攻击者在网页中注入恶意脚本（如 `<script>` 标签），当用户浏览页面时执行，从而窃取 cookie、会话信息、劫持账号等。

**防御方法：**

* 对用户输入进行**严格过滤和转义（HTML Encode）**
* 使用 **Content Security Policy (CSP)** 限制脚本执行来源
* 设置 `HttpOnly` 标记 cookie，防止被 JS 访问
* 避免使用 `innerHTML` 动态插入用户输入内容

---

## ✅ 2. **SQL 注入**

**攻击原理：**
通过构造恶意 SQL 语句注入数据库执行，可能绕过登录、泄露数据或破坏数据库。

**防御方法：**

* 使用 **预编译语句（PreparedStatement）** 或 ORM 框架
* 严格校验用户输入（白名单）
* 数据库权限最小化（如只允许执行查询语句的账号）
* 禁止拼接 SQL 字符串

---

## ✅ 3. **CSRF（跨站请求伪造）**

**攻击原理：**
用户登录网站后未退出，攻击者诱导用户访问一个第三方链接发起请求，网站会误以为是用户的真实操作。

**防御方法：**

* 使用 **CSRF Token**
* 验证请求头中的 `Origin` 和 `Referer`
* 对关键操作使用 POST 请求，避免 GET 修改资源
* 对敏感操作添加双重验证（如验证码）

---

## ✅ 4. **点击劫持（Clickjacking）**

**攻击原理：**
通过隐藏 iframe 来欺骗用户点击网页上的按钮或链接，实际上是在操作另一个网站。

**防御方法：**

* 设置响应头：`X-Frame-Options: DENY` 或 `SAMEORIGIN`
* 使用 Content Security Policy 中的 `frame-ancestors` 限制嵌套来源
* 页面内嵌入防点击脚本检测是否为顶层窗口

---

## ✅ 5. **文件上传漏洞**

**攻击原理：**
上传恶意脚本文件（如 `.php`、`.jsp`），服务器解析后可远程执行代码。

**防御方法：**

* 限制上传文件的 **类型和扩展名**
* 上传文件重命名处理（避免用户指定的文件名生效）
* 存储文件在非 Web 根目录（防止直接访问执行）
* 对上传内容做病毒扫描

---

## ✅ 6. **目录遍历**

**攻击原理：**
通过输入 `../` 等字符访问服务器上敏感目录（如 `/etc/passwd`、`WEB-INF` 等）。

**防御方法：**

* 对路径输入进行规范化处理与校验
* 禁止用户访问不该暴露的目录
* 使用 Web 容器权限隔离

---

## ✅ 7. **敏感信息泄露**

**攻击原理：**
源代码、日志、调试信息或接口暴露敏感内容（如密码、Token、路径结构等）。

**防御方法：**

* 禁止在生产环境输出调试信息
* 不要将配置文件（如 `.env`、`.git`）暴露在 web 根目录下
* 加强日志管理，不记录敏感信息
* 对 API 返回进行脱敏处理

---

## ✅ 8. **暴力破解**

**攻击原理：**
使用程序自动化尝试用户名密码组合进行暴力登录。

**防御方法：**

* 登录接口加验证码（如滑块、人机验证）
* 设置登录失败次数限制、延时处理
* 多因子认证（2FA）

---

## ✅ 9. **中间人攻击（MITM）**

**攻击原理：**
攻击者拦截和篡改用户和服务器之间的数据通信。

**防御方法：**

* **强制使用 HTTPS（TLS）**
* 设置 `Strict-Transport-Security` 响应头
* 不在客户端存储敏感信息（如 token 明文）

---

## ✅ 10. **API 接口未授权访问**

**攻击原理：**
攻击者调用接口时未经过权限校验，能访问本不属于自己的资源。

**防御方法：**

* 所有接口进行严格的**权限校验**
* 后端不能只依赖前端传参控制访问
* 用户身份认证与授权要分离处理
