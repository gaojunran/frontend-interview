---
title: null 和 undefined 的区别是什么？
---

在 JavaScript 中，`null` 和 `undefined` 都表示“无值”或“空”的含义，但它们在语义和使用场景上有明显区别：

---

### 🔹 一、定义与含义

| 值           | 含义               | 来源              |
| ----------- | ---------------- | --------------- |
| `undefined` | 表示**变量声明了但未赋值**  | JavaScript 自动赋值 |
| `null`      | 表示**有意赋值为空对象引用** | 由开发者手动赋值，表示“空值” |

---

### 🔹 二、使用场景区别

* **`undefined`** 通常由 JavaScript 引擎自动赋值，例如：

  ```javascript
  let a;
  console.log(a); // undefined（声明了但没赋值）
  ```

* **`null`** 是开发者主观赋值的，通常表示：

  > “我故意将这个变量设置为空，表示它目前没有值，但未来可能会有”

  ```javascript
  let obj = null; // 表示目前为空对象，稍后再赋值
  ```

---

### 🔹 三、类型区别

```javascript
typeof undefined === "undefined"
typeof null === "object" // 这是历史遗留的 bug，但已成为规范
```

---

### 🔹 四、与 == 和 === 的比较

| 比较                   | 结果      | 原因说明          |
| -------------------- | ------- | ------------- |
| `null == undefined`  | `true`  | 宽松相等认为它们“都为空” |
| `null === undefined` | `false` | 严格相等比较类型不同    |

---

### 🔹 五、常见场景总结

| 场景               | 推荐使用            |
| ---------------- | --------------- |
| 变量声明但未初始化        | `undefined`（自动） |
| 函数参数未传           | `undefined`（默认） |
| 函数无返回值           | `undefined`     |
| 主动清空对象或初始化为空对象引用 | `null`（开发者手动）   |

---

### ✅ 示例

```javascript
function getUser(id) {
  if (!id) return null; // 主动返回“空用户”
}

let user;
console.log(user); // undefined（尚未赋值）

user = null;
console.log(user); // null（已明确置空）
```

---

### ✅ 总结

| 对比点           | `undefined`   | `null`     |
| ------------- | ------------- | ---------- |
| 类型            | `"undefined"` | `"object"` |
| 赋值者           | 引擎自动          | 开发者手动      |
| 是否等于对方 (`==`) | `true`        | `true`     |
| 是否全等 (`===`)  | `false`       | `false`    |
| 使用场景          | 变量未赋值、缺省值     | 显式清空、对象占位  |
