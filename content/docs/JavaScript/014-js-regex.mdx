---

## 一、正则表达式是什么？

正则表达式（RegExp）是一种匹配字符串的**模式（Pattern）**，可以用来检索、替换、验证字符串内容。

在 JavaScript 中正则表达式是一个内置对象，可以通过两种方式创建：

```js
// 字面量
const regex1 = /abc/;

// 构造函数
const regex2 = new RegExp('abc');
```

---

## 二、正则表达式的基本语法

### 1. 元字符（常用）

| 符号      | 含义                    |   |
| ------- | --------------------- | - |
| `.`     | 匹配任意字符（换行符除外）         |   |
| `\d`    | 匹配数字 `[0-9]`          |   |
| `\w`    | 匹配单词字符 `[a-zA-Z0-9_]` |   |
| `\s`    | 匹配空白符                 |   |
| `^`     | 匹配开头                  |   |
| `$`     | 匹配结尾                  |   |
| `[]`    | 匹配字符集合                |   |
| `()`    | 分组                    |   |
| \`      | \`                    | 或 |
| `{n,m}` | 匹配 n 到 m 次            |   |
| `*`     | 匹配 0 或多次              |   |
| `+`     | 匹配 1 或多次              |   |
| `?`     | 匹配 0 或 1 次            |   |
| `\`     | 转义字符                  |   |

### 2. 标志（flags）

* `g`：全局匹配（global）
* `i`：忽略大小写（ignore case）
* `m`：多行匹配（multiline）
* `u`：支持 Unicode
* `y`：粘性匹配（sticky）

---

## 三、常用方法

### 1. RegExp 实例方法

```js
regex.test(str);   // 返回 true/false
regex.exec(str);   // 返回匹配信息（数组）
```

### 2. 字符串方法支持正则：

```js
str.match(regex);
str.replace(regex, replacement);
str.search(regex);
str.split(regex);
```

---

## 四、常见应用场景

### 1. 表单校验

* 手机号校验：

```js
/^1[3-9]\d{9}$/.test('13812345678')
```

* 邮箱校验：

```js
/^[\w.-]+@[\w.-]+\.[a-zA-Z]{2,}$/.test('test@example.com')
```

### 2. 字符串提取/分割

* 提取数字：

```js
'abc123def456'.match(/\d+/g); // ['123', '456']
```

* 按多个标点符号分割：

```js
'hello,world|foo.bar'.split(/[,\|.]/); // ['hello', 'world', 'foo', 'bar']
```

### 3. 文本替换

* 替换敏感词：

```js
content.replace(/(badword1|badword2)/gi, '***');
```

* 驼峰转下划线：

```js
str.replace(/([A-Z])/g, '_$1').toLowerCase();
```

### 4. 高亮搜索词

```js
text.replace(new RegExp(keyword, 'gi'), match => `<mark>${match}</mark>`);
```

---

## 五、注意点

* 使用 `RegExp` 构造函数时，需对 `\` 进行转义。

  ```js
  new RegExp("\\d+"); // 相当于 /\d+/
  ```
* 正则性能对复杂场景需谨慎设计，如“回溯爆炸”。
* 正则可读性差，适合中短匹配任务，复杂匹配可结合代码逻辑处理。

---

## 六、总结

JavaScript 的正则表达式非常强大，在 **字符串处理、表单校验、数据清洗、文本提取** 等方面非常常用。熟练掌握正则可以极大提升开发效率，但也要注意它的可维护性和性能影响。
